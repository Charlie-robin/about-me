What is a Null Pointer?

In my recent travels into the land of C and low level programming. I came across NULL in C. In C, NULL is simply a pointer to memory address 0x000000, which is reserved by the Operating System. Hence throwing an error. I would assume C catches this error itself as an attempt to access forbidden memory would typically cause a program to be killed by the host OS. I just find it interesting that in some languages these etherial ever-looming concepts become extremly tangible.

Why do we end lines in semicolons?

I've not been able to find very much evidence for this, but I've been playing around with a bit of x86 assembly lately (for understanding, not real projects), and seeing some of my code look like this:

mv aa, 0x005000 ; move a value into something
cmp aa, 0x000000 ; compare aa against someting
lt myLabel ; go to another subroutine

This is a common pattern in assembly, semicolons are used for comments, and most lines end in these comments due to the nature of the language. My theory is that this naturally translated into higher-level languages, as the creators of most modern high-level language will have gotten their start in the realm of low-level.
